# 用户接入指南

本文档面向使用 Cursor、Claude Desktop、Kimi Code 等 AI Agent 的用户，说明如何接入中央知识库。

---

## 1. 获取 API Key

1. 联系团队管理员，说明需要访问知识库的用途
2. 管理员会在后台为你创建 API Key，并告知你具体的 Key 值
3. **API Key 仅在创建时展示一次，请务必妥善保存，不要泄露给他人**

### Key 权限说明

| 权限 | 说明 |
|------|------|
| `read` | 只能检索知识库，不能修改内容 |
| `write` | 可以添加、更新、删除文档；如需同时检索还必须同时授予 `read` |

> 普通用户通常只需要 `read` 权限即可。如需向知识库贡献文档，可申请 `write` 权限。

## 2. Cursor 配置

Cursor 支持 MCP StreamableHTTP 协议，可直接通过 URL 配置。

### 配置步骤

1. 打开 Cursor Settings → MCP
2. 点击 **Add new MCP server**
3. 填写以下配置：

```json
{
  "mcpServers": {
    "knowledge-base": {
      "url": "http://服务器地址/mcp",
      "headers": {
        "X-API-Key": "sk-你的API Key"
      }
    }
  }
}
```

### 连接地址说明

| 部署模式 | 地址示例 |
|---------|---------|
| Docker 内网 IP | `http://192.168.1.100/mcp` |
| Docker 主机名 | `http://KB-SERVER/mcp` |
| Docker mDNS 域名 | `http://KB-SERVER.local/mcp` |
| 本地开发 | `http://localhost:8000/mcp` |
| 外网 HTTPS | `https://wiki.yourcompany.com/mcp` |

> 提示：Windows 局域网使用**主机名**连接最方便，DHCP 换 IP 也无需改配置。在客户端 `ping KB-SERVER` 确认能通即可。

## 3. Claude Desktop 配置

若所用 Claude Desktop 版本支持远程 Streamable HTTP，请直接使用第 2 节的 URL + Header 配置。只有客户端版本仅支持 stdio 时，才需要使用兼容代理把 `/sse` 转为 stdio；代理的参数格式以其当前文档为准。

### 配置步骤

1. 打开 Claude Desktop Settings → Developer → Edit Config
2. 编辑 `claude_desktop_config.json`：

```json
{
  "mcpServers": {
    "knowledge-base": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-proxy",
        "http://服务器地址/sse?api_key=sk-你的API Key"
      ]
    }
  }
}
```

3. 保存文件后重启 Claude Desktop

### 内网地址示例

```json
{
  "mcpServers": {
    "knowledge-base": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-proxy",
        "http://192.168.1.100/sse?api_key=sk-示例Key请替换"
      ]
    }
  }
}
```

> URL 查询参数中的 Key 可能进入日志和历史记录。客户端/代理支持 Header 时必须优先使用 `X-API-Key`；示例仅用于无法传 Header 的旧式适配器。

## 4. Kimi Code 配置

Kimi Code 支持 MCP StreamableHTTP 协议，配置方式与 Cursor 类似。

### 配置步骤

1. 打开 Kimi Code 设置 → MCP 服务器
2. 添加新的 MCP 服务器：

```json
{
  "mcpServers": {
    "knowledge-base": {
      "url": "http://服务器地址/mcp",
      "headers": {
        "X-API-Key": "sk-你的API Key"
      }
    }
  }
}
```

> 注意：Kimi Code 的配置文件位置因版本而异，请参考 Kimi Code 官方文档确认具体路径。

## 5. Windsurf / 其他 MCP 客户端

支持 MCP 协议的客户端均可接入，根据客户端支持的协议类型选择合适的端点：

- **支持 StreamableHTTP**：使用 `/mcp` 端点，配置 URL + Headers
- **仅支持 stdio**：使用 `mcp-proxy` 中转，配置 command + args

### Windsurf 配置示例

```json
{
  "mcpServers": {
    "knowledge-base": {
      "url": "http://服务器地址/mcp",
      "headers": {
        "X-API-Key": "sk-你的API Key"
      }
    }
  }
}
```

## 6. REST API 直接调用

不通过 MCP 协议，直接通过 HTTP 请求调用知识库接口。

```bash
# 检索知识库（GET，查询参数为 q）
curl "http://服务器地址/api/search?q=搜索关键词&top_k=5" \
  -H "X-API-Key: sk-你的API Key"

# 列出文档
curl "http://服务器地址/api/documents?limit=20&offset=0" \
  -H "X-API-Key: sk-你的API Key"

# 查看目录树
curl "http://服务器地址/api/directories" \
  -H "X-API-Key: sk-你的API Key"
```

REST 当前不提供“获取全文”端点；需要全文时使用 MCP `get_document`。Windows 原生开发模式把上述地址改为 `http://服务器地址:8000/...`。

## 7. 常用 MCP 工具速查

接入后，你可以在 Agent 中使用以下工具：

| 工具 | 用途 | 必需参数 |
|------|------|---------|
| `search_knowledge` | 向量、关键词、结构和可选图谱关联混合检索 | `query` |
| `get_document` | 获取文档完整内容 | `doc_id` |
| `list_documents` | 列出文档（支持分页） | — |
| `list_directories` | 查看目录树 | — |
| `add_document` | 添加新文档（需 write 权限） | `title`, `content` |
| `update_document` | 更新文档（需 write 权限） | `doc_id`, `title`, `content` |
| `delete_document` | 删除文档（需 write 权限） | `doc_id` |

## 8. 常见问题

### Q: 连接失败或超时

1. Docker 部署确认服务器是否在线：浏览器访问 `http://服务器地址/health`
2. 确认 80（HTTP）或 443（HTTPS）端口可达；只有 Windows 原生模式使用 8000
3. Windows 防火墙、路由器或云安全组可能阻挡端口，检查入站规则
4. 使用主机名连接时，确保客户端和服务器在同一网段

### Q: API Key 过期了怎么办？

联系管理员续期或重新创建 Key。过期后知识库将无法访问，需更新配置文件中的 Key。

### Q: 提示"权限不足"

你的 API Key 是 `read` 权限但尝试了写入操作（add_document 等）。联系管理员申请 `write` 权限。

### Q: 检索结果不准确

- 尝试换用更精准的关键词
- 使用 `filter_tags` 或 `filter_path` 缩小搜索范围
- 增大 `top_k` 获取更多候选结果
- 如果管理员刚更新了很多文档，请确认知识图谱已重新构建；图谱结果可通过 `channel=graph` 识别

### Q: 返回 503“搜索请求繁忙”

按响应中的 `retry_after_ms` 等待后重试，并加入少量随机抖动。持续出现时由管理员检查硬件档位、`SEARCH_MAX_CONCURRENCY`、Ollama 并行数和机器资源。

### Q: Claude Desktop 报 mcp-proxy 未找到

确保 Node.js 已安装（`node --version`），首次运行会自动下载 `mcp-proxy`。如网络受限，可手动安装：

```bash
npm install -g mcp-proxy
```

然后将配置改为：

```json
{
  "command": "mcp-proxy",
  "args": ["http://服务器地址/sse?api_key=sk-你的API Key"]
}
```

### Q: 如何验证连接是否正常？

在 Agent 中直接提问："帮我搜索知识库中关于 XX 的内容"，如果返回了结果则连接正常。也可通过健康检查接口验证：

```bash
curl http://服务器地址/health
```

返回 `{"status": "ok"}` 表示服务正常运行。
