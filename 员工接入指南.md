# 员工接入指南

本文档面向使用 Cursor、Claude Desktop、Kimi Code 等 AI Agent 的员工，说明如何接入公司中央知识库。

---

## 1. 获取 API Key

1. 联系团队管理员，说明需要访问知识库的用途
2. 管理员会在后台为你创建 API Key，并告知你具体的 Key 值
3. **API Key 仅在创建时展示一次，请务必妥善保存，不要泄露给他人**

### Key 权限说明

| 权限 | 说明 |
|------|------|
| `read` | 只能检索知识库，不能修改内容 |
| `write` | 可以检索、添加、更新、删除文档 |

> 普通员工通常只需要 `read` 权限即可。如需向知识库贡献文档，可申请 `write` 权限。

---

## 2. Cursor 配置

### 方式一：直接 SSE 连接（推荐）

1. 打开 Cursor Settings（设置）
2. 进入 MCP 配置页面
3. 添加新的 MCP Server：

```json
{
  "mcpServers": {
    "knowledge-base-management": {
      "url": "https://kb.yourcompany.com/sse",
      "headers": {
        "X-API-Key": "sk-your-api-key-here"
      }
    }
  }
}
```

4. 保存后，Cursor 会自动连接知识库

### 方式二：使用 mcp-proxy（如果直接连接不稳定）

```json
{
  "mcpServers": {
    "knowledge-base-management": {
      "command": "npx",
      "args": ["-y", "mcp-proxy", "https://kb.yourcompany.com/sse?api_key=sk-your-api-key-here"]
    }
  }
}
```

## 3. Claude Desktop 配置

编辑配置文件：
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "knowledge-base-management": {
      "command": "npx",
      "args": ["-y", "mcp-proxy", "https://kb.yourcompany.com/sse?api_key=sk-your-api-key-here"]
    }
  }
}
```

重启 Claude Desktop 生效。

## 4. Kimi Code / 其他支持 MCP 的 IDE

参考各自 IDE 的 MCP 配置文档，填入以下信息：

| 配置项 | 值 |
|--------|-----|
| 名称 | Knowledge Base Management |
| 类型 | SSE |
| URL | `https://kb.yourcompany.com/sse` |
| Header | `X-API-Key: sk-your-api-key-here` |

## 5. 使用知识库

配置成功后，在你的 Agent 对话中可以直接使用知识库工具：

### 检索知识

直接提问即可，Agent 会自动调用知识库检索：

> "我们公司的 API 认证流程是什么？"
> "查找关于微服务拆分的文档"

### 添加文档（需要 write 权限）

> "帮我将以下内容添加到知识库的技术/API目录下：[你的 Markdown 内容]"

Agent 会调用 `add_document` 或 `import_markdown` 工具完成添加。

## 6. 常见问题

### Q: 连接失败或提示 403？

A: 检查以下几点：
1. API Key 是否正确复制，不要有多余空格
2. Key 是否已过期或被吊销，联系管理员确认
3. 公司网络是否需要代理才能访问知识库服务器

### Q: 检索结果不准确？

A: 尝试以下方式：
1. 使用更具体的关键词
2. 指定目录路径，如 "在技术/API目录下查找..."
3. 确认相关文档已经导入知识库

### Q: 添加文档时提示 423 Locked？

A: 表示当前有其他人在写入知识库，Chroma 不支持并发写入。稍等几秒后重试即可。

### Q: 我的 API Key 过期了怎么办？

A: 联系管理员重新申请新的 API Key。

---

## 7. 注意事项

- **API Key 属于个人凭证，请勿分享给他人**
- **通过 Agent 添加的文档全公司可见，请注意不要上传敏感信息**
- 如离职或转岗，请及时通知管理员吊销你的 API Key
- 建议定期备份自己添加的重要文档原文
