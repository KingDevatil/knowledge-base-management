新增部署模式

用于区分不同部署方案，例如“内网开发”“外网生产”。只影响后台显示，不会写入运行配置。

内网模式适合局域网访问；外网模式会启用外网域名相关配置；内外网混合会同时保留内网和外网访问参数。

允许跨域访问的来源。开发环境可填 *；生产环境建议填写 https://你的外网域名，多个来源用英文逗号分隔。

内网访问时使用的计算机名、域名或 IPv4；多个值用英文逗号分隔。

外网模式或内外网混合模式必填，例如 kb.example.com。保存并启用后会写入 EXTERNAL_DOMAIN。

HTTPS 证书链文件路径。推荐使用项目相对路径或容器内挂载路径，例如 certs/fullchain.pem 或 /etc/letsencrypt/live/kb/fullchain.pem。留空表示由反向代理或外部网关处理证书。

HTTPS 证书私钥文件路径。推荐使用项目相对路径或容器内挂载路径，例如 certs/privkey.pem 或 /etc/letsencrypt/live/kb/privkey.pem。请确保服务运行账号有读取权限。

新增反向代理

生成 Caddy 或 Nginx 反代配置，将域名请求转发到当前服务。

用于区分不同反代方案，例如“Caddy 外网”“Nginx 生产”。

Caddy 可自动申请证书；Nginx 适合已有网关，通常需要手动配置证书。

启用状态用于后台标记当前方案，实际生效还需要将生成的配置应用到 Caddy 或 Nginx 并 reload。

公网访问使用的域名，必须已通过 DNS 或 DDNS 解析到当前网络。

当前知识库后台服务端口，默认是 8000。

反向代理访问应用的内网地址，本机部署通常填写 127.0.0.1。

Caddy 会优先自动签发证书；Nginx 需要证书和私钥路径都已配置。

Docker 内置 Nginx 会从 nginx/ssl 目录读取相对路径，例如 你的域名/fullchain.pem；也可填写容器内绝对路径。

与证书文件配套的私钥路径。相对路径同样从 nginx/ssl 目录解析，例如 你的域名/privkey.pem。

保存时如果留空会自动生成；如果手动修改，系统会保留你编辑后的文本。
